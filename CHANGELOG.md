# Changelog

本项目遵循语义化版本。功能提交保留在 Git `main` 分支。

## 0.4.0

- 增加可配置裁判并发度。
- 增加线程安全的调用预算预留，防止并发突破调用次数上限。
- 增加 pending result 持久化，中断后复用已付费裁判结果。
- 并行获取判断、按持久化赛程串行更新 Elo，保证结果确定性。
- token 或金额预算下限制单并发，避免无法预估的并行超支。

## 0.3.0

- 增加持久化瑞士轮赛程，优先相近 Elo 且避免重复对手。
- 增加双向裁判；方向不一致时标记争议并按平局计分。
- 增加本地近重复检测，自动补生成缺失候选。
- 增加版本化模型定价、实际 token 金额估算与金额预算。
- 默认配对由随机改为瑞士轮，默认计划调用量由 35 降至 20。

## 0.2.0

- 增加可版本化、自定义权重的评分 rubric。
- 增加分维度评分、置信度、平局和程序侧加权判胜。
- 增加 LLM calls/token 用量统计及预算安全停止。
- 增加失败或预算停止会话续跑，避免重复计分和重复进化。
- 增加包含排行榜、谱系、rubric 和逐场证据的 Markdown 报告。

## 0.1.0

- 建立终端 MVP：生成、配对、LLM 裁判、Elo、进化、JSON 存储和排行榜。
- 增加调用量估算、基础重试、输入校验、原子保存和无网络测试。
